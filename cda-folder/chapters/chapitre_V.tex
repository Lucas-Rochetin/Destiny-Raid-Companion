\section{Architecture 3 tiers}

L'architecture Destiny Raid Companion suit une approche 3-tiers classique avec séparation nette entre présentation, logique métier et données. Cette séparation permet une maintenance simplifiée, une scalabilité indépendante de chaque couche et une meilleure testabilité.

\subsection{Couche Présentation (Frontend)}

Le frontend est développé avec \textbf{Next.js 14} et TypeScript.

\subsubsection*{Pourquoi ces choix techniques ?}

\begin{itemize}
    \item \textbf{Next.js 14 :} Je bénéficie du Server-Side Rendering pour les guides, ce qui les rend indexables par les moteurs de recherche et plus rapides à l'affichage.
    \item \textbf{TypeScript :} J'évite les erreurs de type au développement.
    \item \textbf{TailwindCSS :} Je stylise sans quitter mon HTML.
\end{itemize}

\subsection{Couche Logique Métier (Backend)}

Le backend utilise Node.js avec Express.js, structuré selon le pattern Controller-Service-Repository.

\subsubsection*{Pourquoi cette organisation ?}

Séparer les responsabilités rend le code plus clair et plus facile à faire évoluer. Si je dois changer une règle métier, je sais exactement où la trouver.

\subsubsection*{Exemple : le contrôleur}

\begin{lstlisting}[language=JavaScript]
const createSquad = async (req, res, next) => {
  try {
    const validation = squadSchema.safeParse(req.body);
    if (!validation.success) {
      return res.status(400).json({
        error: 'Données invalides'
      });
    }

    const squad = await this.squadService.createSquad(
      validation.data, 
      req.user.id
    );

    res.status(201).json({
      success: true,
      data: squad
    });
  } catch (error) {
    next(error);
  }
};
\end{lstlisting}

\subsubsection*{Exemple : le service}

\begin{lstlisting}[language=JavaScript]
class SquadService {
  async createSquad(squadData, leaderId) {
    if (squadData.maxMembers > 12) {
      throw new BusinessError('Une escouade ne peut pas dépasser 12 membres');
    }

    const userSquadCount = await this.squadRepository.countByUser(leaderId);
    if (userSquadCount >= 5) {
      throw new BusinessError('Limite de 5 escouades par utilisateur atteinte');
    }

    return await this.db.transaction(async (trx) => {
      const squad = await this.squadRepository.create(trx, {
        ...squadData,
        leader_id: leaderId
      });

      await this.squadMemberRepository.create(trx, {
        squad_id: squad.id,
        user_id: leaderId,
        role: 'leader'
      });

      return squad;
    });
  }
}
\end{lstlisting}

\subsubsection*{Exemple : le repository}

\begin{lstlisting}[language=JavaScript]
class SquadRepository {
  async create(trx, squadData) {
    return await trx.squad.create({
      data: {
        name: squadData.name,
        description: squadData.description,
        leader_id: squadData.leader_id,
        max_members: squadData.maxMembers
      }
    });
  }

  async countByUser(userId) {
    return await this.db.squad.count({
      where: {
        OR: [
          { leader_id: userId },
          { members: { some: { user_id: userId } } }
        ]
      }
    });
  }
}
\end{lstlisting}

\subsection{Couche Données (Database)}

Une architecture multi-base optimisée pour chaque type d'usage.

\begin{itemize}
    \item \textbf{PostgreSQL :} Données transactionnelles (utilisateurs, escouades, sessions)
    \item \textbf{Redis :} Cache sessions, données d'API Bungie
    \item \textbf{Prisma :} ORM pour PostgreSQL avec migrations versionnées
\end{itemize}

\subsubsection*{Schéma Prisma}

\begin{lstlisting}[language=JavaScript]
model User {
  id           Int      @id @default(autoincrement())
  bungie_id    String   @unique
  display_name String
  created_at   DateTime @default(now())
  
  squads_led   Squad[]  @relation("SquadLeader")
  squad_members SquadMember[]
  sessions     Session[]
}

model Squad {
  id          Int      @id @default(autoincrement())
  name        String
  description String?
  leader_id   Int
  created_at  DateTime @default(now())
  max_members Int      @default(6)
  
  leader      User     @relation("SquadLeader", fields: [leader_id], references: [id])
  members     SquadMember[]
  sessions    Session[]
}
\end{lstlisting}

\subsection{Avantages de l'architecture 3 tiers}

\begin{itemize}
    \item \textbf{Séparation des responsabilités :} Chaque couche a un rôle unique.
    \item \textbf{Scalabilité indépendante :} Je peux faire évoluer chaque couche séparément.
    \item \textbf{Maintenabilité :} Les modifications sont isolées.
    \item \textbf{Sécurité :} Authentification centralisée.
    \item \textbf{Testabilité :} Tests unitaires par couche facilités.
\end{itemize}
